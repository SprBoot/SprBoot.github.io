DeepSort源码解析二

外观特征余弦距离计算

创建数组R存储128维度的特征值，数组序号为多目标跟踪的id，数组内容为视频帧的特征值。数组的长度不超过100，即存储的目标特征帧不超过100帧。

余弦相似度计算公式

这里的向量A为i维的，即（a1,a2,a3,…,ai）

向量b也为i维，即（b1,b2,b3,…,bi）

余弦距离为1-cosθ。

DeepSort中使用数组R中的外观特征与检测框的特征计算余弦相似度，取最小值记为C。

马氏距离

在使用欧氏距离度量变量之间的相关程度时，

这里x,y为二维，若存在多维的情况下，当变量之间差距过大，实例之间差距较小。往往相关程度受大变量的影响造成误差。

（身高/cm，体重/g）

已知小明（160,60000）；小王（160,59000）；小李（170，60000）。根据常识可以知道小明和小王体型相似。但是如果根据欧几里得距离来判断，小明和小王的距离要远远大于小明和小李之间的距离，即小明和小李体型相似。这是因为不同特征的度量标准之间存在差异而导致判断出错。以克（g）为单位测量人的体重，数据分布比较分散，即方差大，而以厘米为单位来测量人的身高，数据分布就相对集中，方差小。马氏距离的目的就是把方差归一化，使得特征之间的关系更加符合实际情况。

即马氏距离引入了协方差矩阵。

马氏距离的计算公式

使用马氏距离对余弦距离进行筛选

d为检测框的位置，y为预测框的位置，S为检测框与预测框的协方差矩阵。

DeepSORT中，由马氏距离计算获得运动特征提供了物体定位的可能信息，在短期预测中非常有效。由余弦距离计算获得外观特征在目标被长期遮挡后，恢复目标的ID编号，减少ID切换次数，比较有效。通过加权运算，结合两个特征

针对每一个检测器都会分配一个跟踪器，每个跟踪器会设定一个time_since*_*update参数。如果跟踪器完成匹配并进行更新，该参数会重置为0，否则会+1。实际上，级联匹配换句话说就是不同优先级的匹配。在级联匹配中，会根据这个参数来对跟踪器分先后顺序，参数小的先匹配，参数大的后匹配。

匈牙利匹配算法

理解与计算可参考这篇文章

在DeepSort中使用匈牙利算法进行匹配主要用于检测框与结果B（预测框）二分图匹配，最后获取最多没有公共端点的边。

经过匈牙利算法之后，B结果自然而然的就分为了匹配项与未匹配项。

        def gated_metric(tracks, dets, track_indices, detection_indices):
            features = np.array([dets[i].feature for i in detection_indices])
            targets = np.array([tracks[i].track_id for i in track_indices])
            cost_matrix = self.metric.distance(features, targets)
            cost_matrix = linear_assignment.gate_cost_matrix(
                self.kf, cost_matrix, tracks, dets, track_indices,
                detection_indices)

            return cost_matrix

        # 区分确定态与不确定态的轨迹
        confirmed_tracks = [
            i for i, t in enumerate(self.tracks) if t.is_confirmed()]
        unconfirmed_tracks = [
            i for i, t in enumerate(self.tracks) if not t.is_confirmed()]

        # 确定态与检测框的级联匹配
        matches_a, unmatched_tracks_a, unmatched_detections = \
            linear_assignment.matching_cascade(
                gated_metric, self.metric.matching_threshold, self.max_age,
                self.tracks, detections, confirmed_tracks)

        # 不确定态与匹配失败次数为1的进行IOU匹配
        iou_track_candidates = unconfirmed_tracks + [
            k for k in unmatched_tracks_a if
            self.tracks[k].time_since_update == 1]
        unmatched_tracks_a = [
            k for k in unmatched_tracks_a if
            self.tracks[k].time_since_update != 1]
        matches_b, unmatched_tracks_b, unmatched_detections = \
            linear_assignment.min_cost_matching(
                iou_matching.iou_cost, self.max_iou_distance, self.tracks,
                detections, iou_track_candidates, unmatched_detections)

        # 计算两次计算的结果和并去除重复项
        matches = matches_a + matches_b
        unmatched_tracks = list(set(unmatched_tracks_a + unmatched_tracks_b))
        return matches, unmatched_tracks, unmatched_detections
